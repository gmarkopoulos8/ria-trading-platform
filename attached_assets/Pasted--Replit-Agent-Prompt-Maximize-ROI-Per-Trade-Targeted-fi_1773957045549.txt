## BUG 1 — Entry Price Uses TOS When Trading on Alpaca (Critical)

**File**: `server/src/services/autotrader/AutoTradeExecutor.ts`

`resolveEntryPrice()` falls back to `tosInfoService.getQuotes()` for stocks — but TOS may not be connected when running Alpaca paper trades. When TOS isn't connected this returns 0, and the trade is rejected with "Could not resolve entry price."

Find the full `resolveEntryPrice` function and replace it:

```typescript
async function resolveEntryPrice(signal: AutoTradeSignal): Promise<number> {
  // If the scan already gave us a price, use it
  if (signal.entryPrice && signal.entryPrice > 0) return signal.entryPrice;

  const exchange = signal.exchange ?? 'PAPER';

  try {
    if (signal.assetClass === 'crypto' || signal.assetClass === 'CRYPTO') {
      const { getAssetPrice } = await import('../hyperliquid/hyperliquidInfoService');
      const price = await getAssetPrice(signal.symbol);
      if (price && price > 0) return price;
    }

    // For PAPER exchange, use Alpaca's own quote — never depend on TOS being connected
    if (exchange === 'PAPER') {
      try {
        const { getLatestQuote } = await import('../alpaca/alpacaInfoService');
        const quote = await getLatestQuote(signal.symbol);
        if (quote && quote > 0) return quote;
      } catch {
        // fall through to Yahoo Finance
      }
    }

    // For TOS exchange, use TOS quotes
    if (exchange === 'TOS') {
      try {
        const { getQuotes } = await import('../tos/tosInfoService');
        const quotes = await getQuotes([signal.symbol]);
        const q = quotes[signal.symbol];
        if (q) return q.lastPrice ?? q.mark ?? q.askPrice ?? 0;
      } catch {
        // fall through
      }
    }

    // Universal fallback — Yahoo Finance via StocksService (always available, no auth needed)
    const { stocksService } = await import('../market/stocks/StocksService');
    const quote = await stocksService.quote(signal.symbol);
    if (quote?.price && quote.price > 0) return quote.price;
  } catch {
    // last resort below
  }

  return 0;
}
```

Also add `getLatestQuote` to `server/src/services/alpaca/alpacaInfoService.ts`. Add this function at the bottom:

```typescript
export async function getLatestQuote(symbol: string): Promise<number | null> {
  // Alpaca data API provides real-time quotes
  try {
    const headers = authHeaders();
    const url = `https://data.alpaca.markets/v2/stocks/${symbol}/quotes/latest`;
    const { data } = await axios.get(url, { headers, timeout: 5000 });
    const bid = parseFloat(data?.quote?.bp ?? '0');
    const ask = parseFloat(data?.quote?.ap ?? '0');
    if (bid > 0 && ask > 0) return (bid + ask) / 2; // mid price
    return null;
  } catch {
    return null;
  }
}
```

## ROI FIX 3 — Auto-Monitor Runs Automatically Every 5 Minutes

**File**: `server/src/index.ts`

Currently exits only happen when you manually click "Check Exits". If a position hits its stop while you're away, it sits there bleeding. Add an automatic monitor loop.

Add this import near the other service imports:
```typescript

import { closePosition } from './services/alpaca/alpacaExchangeService';
import { getCurrentParams } from './services/alpaca/AdaptiveParameterEngine';

```

Add this function and call it after `startAdaptiveLoop()`:

```typescript
// Auto-monitor: check stop/target every 5 minutes during market hours
function startAlpacaAutoMonitor() {
  setInterval(async () => {
    try {
      if (!hasAlpacaCredentials() || isPauseActive() || isKillswitchActive()) return;

      const nyHour = parseInt(
        new Date().toLocaleString('en-US', { timeZone: 'America/New_York', hour: 'numeric', hour12: false }), 10
      );
      if (nyHour < 9 || nyHour >= 16) return; // market hours only

      const positions = await getPositions();
      if (!positions || positions.length === 0) return;

      // Use adaptive params if available, otherwise safe defaults
      // We don't have userId here so use conservative defaults
      // Individual user adaptive params are applied when they manually trigger
      const defaultStop   = 4.0;
      const defaultTarget = 8.0;

      for (const pos of positions) {
        const entry   = parseFloat((pos as any).avg_entry_price ?? '0');
        const current = parseFloat((pos as any).current_price   ?? '0');
        if (entry <= 0 || current <= 0) continue;

        const pnlPct = ((current - entry) / entry) * 100;

        if (pnlPct <= -defaultStop || pnlPct >= defaultTarget) {
          const reason = pnlPct <= -defaultStop
            ? `Auto stop-loss: ${pnlPct.toFixed(2)}%`
            : `Auto take-profit: +${pnlPct.toFixed(2)}%`;

          console.info(`[AutoMonitor] Closing ${(pos as any).symbol} — ${reason}`);

          await closePosition((pos as any).symbol).catch((e: any) =>
            console.warn(`[AutoMonitor] Close failed for ${(pos as any).symbol}:`, e?.message)
          );

          // Write PnL back
          try {
            const exitPrice = current;
            const marketValue = parseFloat((pos as any).market_value ?? '0');
            const dollarPnl   = pnlPct / 100 * Math.abs(marketValue);
            await prisma.autoTradeLog.updateMany({
              where: { symbol: (pos as any).symbol, exchange: 'PAPER', status: 'FILLED', exitPrice: null },
              data:  { exitPrice, pnl: dollarPnl, status: 'CLOSED' },
            });
          } catch { /* ignore writeback errors */ }
        }
      }
    } catch (err: any) {
      // Swallow all errors — this is a background loop
    }
  }, 5 * 60_000); // every 5 minutes
}

startAlpacaAutoMonitor();
```

Also add the prisma import at the top of `index.ts` if not already there:

## SUMMARY OF CHANGES

### Bugs Fixed
1. `resolveEntryPrice` — Alpaca now uses its own quote API, not TOS. Trades won't fail when TOS is disconnected.
2. PnL writeback — when monitor closes a position, exit price and dollar PnL are now written to `autoTradeLog` so the adaptive engine has data.
3. Circuit breaker — Alpaca killswitch and pause state now correctly block trades.

### ROI Improvements
4. Signal ranking — sorted by `compositeScore` (best overall setup quality) not just `convictionScore`
5. Bull trend aggression — size +15%, target +25%, conviction bar -3 in confirmed bull markets
6. Winning streak — now triggers at profit factor 1.5 (was 1.8), and rewards are larger (+20% each)
7. Auto-monitor every 5 minutes — positions never sit past their stop/target unattended
8. Default bounds tuned — max position size 1.5x, max target 25%, min target 4%
9. 5x candidate pool — more signals to choose from, re-sorted by composite score
10. Profit factor tracking — raises conviction sooner (1.3 threshold, not 1.0), lowers it on strong performance
