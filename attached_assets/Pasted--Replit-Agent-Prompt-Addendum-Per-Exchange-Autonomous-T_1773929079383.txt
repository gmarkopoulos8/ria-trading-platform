# Replit Agent Prompt — Addendum: Per-Exchange Autonomous Trading Parameters

This is an addendum to the previous autonomous trading prompt. Apply these changes on top of everything already built. Do NOT re-implement anything from the previous prompt — only add and extend what’s described here.

-----

## OVERVIEW

Each exchange integration (Hyperliquid and ThinkorSwim) needs its own independent parameter profile that controls exactly how the autonomous trader behaves on that platform. These parameters must be:

1. **Stored per-user in the database** (not env vars — users configure them through the UI)
1. **Validated on the backend** before any order is placed
1. **Displayed and editable** on each exchange’s existing dashboard page (HyperliquidDashboard and TosDashboard) as a new collapsible “Autonomous Trading Parameters” panel
1. **Enforced in AutoTradeExecutor** before routing to each exchange

-----

## PHASE A — Database Schema

Add to `server/prisma/schema.prisma`:

```prisma
model ExchangeAutoConfig {
  id        String   @id @default(cuid())
  userId    String
  exchange  String   // 'hyperliquid' | 'tos'

  // ── Activation & Scheduling ──────────────────────────────────
  enabled               Boolean  @default(false)
  // How long to run autonomously before auto-pausing (null = indefinite)
  sessionDurationHours  Float?
  // Auto-pause after this wall-clock time each day (e.g. "16:00" ET = market close)
  dailyCutoffTime       String?  // "HH:MM" in exchange's local timezone
  // Days of week to trade (0=Sun … 6=Sat). Empty = all days
  activeDays            Int[]    @default([1,2,3,4,5])
  // Timestamp when current autonomous session started (null if not running)
  sessionStartedAt      DateTime?
  // Auto-pause triggered at this time (set when sessionDurationHours elapses)
  sessionPausedAt       DateTime?

  // ── Capital Allocation ───────────────────────────────────────
  // Maximum total USD this integration is ever allowed to have deployed (hard cap)
  capitalHardLimitUsd   Float    @default(5000)
  // Target allocation — how much to actively deploy in open positions at once
  capitalTargetUsd      Float    @default(2000)
  // Max USD in any single trade/position
  maxPositionSizeUsd    Float    @default(500)
  // Min USD for a position to be worth placing (filters out tiny nano-positions)
  minPositionSizeUsd    Float    @default(50)

  // ── Risk Profile ─────────────────────────────────────────────
  riskMode              String   @default("MODERATE") // CONSERVATIVE | MODERATE | AGGRESSIVE | CUSTOM
  // Risk % per trade (only used when riskMode = CUSTOM)
  customRiskPct         Float?
  // Stop loss floor — never set a stop tighter than this % below entry
  minStopDistancePct    Float    @default(1.0)
  // Stop loss ceiling — if thesis invalidation is wider than this, skip the trade
  maxStopDistancePct    Float    @default(12.0)
  // Minimum reward:risk ratio required to take a trade (e.g. 1.5 = need 1.5:1)
  minRewardRiskRatio    Float    @default(1.5)
  // Maximum drawdown on THIS exchange before auto-pausing (independent of global circuit breaker)
  maxDrawdownPct        Float    @default(5.0)
  // Maximum daily loss in USD on this exchange before auto-pausing
  maxDailyLossUsd       Float    @default(200)

  // ── Position Limits ──────────────────────────────────────────
  maxConcurrentPositions Int     @default(3)
  // Max number of trades to place per scan cycle (throttle)
  maxTradesPerScan      Int      @default(2)
  // Max trades in a single calendar day
  maxTradesPerDay       Int      @default(5)
  // Cooldown between placing orders on this exchange (minutes)
  orderCooldownMinutes  Int      @default(15)

  // ── Signal Thresholds ────────────────────────────────────────
  minConvictionScore    Float    @default(78)
  minConfidenceScore    Float    @default(60)
  // Only trade these biases ('BULLISH', 'BEARISH', or both)
  allowedBias           String[] @default(["BULLISH"])
  // Only act on these recommended actions
  allowedActions        String[] @default(["STRONG_BUY", "BUY"])
  // Minimum thesis health score for HOLDING — auto-exit if health drops below this
  minHoldThesisHealth   Float    @default(35)

  // ── Exchange-Specific (Hyperliquid only) ─────────────────────
  // Default leverage for new positions (1 = spot-equivalent, no leverage)
  defaultLeverage       Int      @default(2)
  // Maximum leverage ever allowed regardless of conviction
  maxLeverage           Int      @default(5)
  // Use cross margin (true) or isolated margin (false)
  useCrossMargin        Boolean  @default(true)
  // Only trade these specific assets (empty = all assets in scan universe)
  allowedAssets         String[] @default([])
  // Never trade these assets even if scan recommends them
  blockedAssets         String[] @default([])
  // Enable short positions (false = long only)
  allowShorts           Boolean  @default(false)

  // ── Exchange-Specific (TOS / Schwab only) ───────────────────
  // Asset types allowed to trade (EQUITY, OPTION, ETF)
  allowedAssetTypes     String[] @default(["EQUITY", "ETF"])
  // Order session: NORMAL | SEAMLESS (pre/after hours)
  orderSession          String   @default("NORMAL")
  // Default order duration
  orderDuration         String   @default("DAY")
  // Enable bracket orders (auto-attach TP and SL to every entry)
  useBracketOrders      Boolean  @default(true)
  // Sector blocklist — never buy stocks in these sectors
  blockedSectors        String[] @default([])
  // Min average daily volume (shares) — skip illiquid stocks
  minAvgDailyVolume     Int      @default(500000)
  // Min market cap in USD — skip micro/nano caps
  minMarketCapUsd       Float    @default(1000000000) // $1B default

  // ── Audit ────────────────────────────────────────────────────
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@unique([userId, exchange])
  @@index([userId])
}
```

Run `npx prisma migrate dev --name add_exchange_auto_config` after adding this model.

## PHASE B — Backend Service

**File**: `server/src/services/autotrader/ExchangeAutoConfigService.ts`

```typescript
export interface ExchangeAutoConfig { /* mirrors the Prisma model */ }

export interface ConfigValidationResult {
  valid: boolean;
  errors: string[];
  warnings: string[];
}
```

Implement these functions:

**`getConfig(userId: string, exchange: 'hyperliquid' | 'tos'): Promise<ExchangeAutoConfig>`**

- Fetches from DB; if no record exists, creates one with all defaults and returns it
- Merge with live exchange state for context (e.g., actual account equity to validate capital limits)

**`saveConfig(userId: string, exchange: string, updates: Partial<ExchangeAutoConfig>): Promise<ExchangeAutoConfig>`**

- Validate before saving (call `validateConfig`)
- Strip fields that don’t belong to the exchange type (e.g., don’t save `defaultLeverage` for TOS)
- Return the saved config

**`validateConfig(config: Partial<ExchangeAutoConfig>, exchange: string): ConfigValidationResult`**

- `capitalTargetUsd` must be ≤ `capitalHardLimitUsd`
- `maxPositionSizeUsd` must be ≤ `capitalTargetUsd`
- `minPositionSizeUsd` must be < `maxPositionSizeUsd`
- `minStopDistancePct` must be < `maxStopDistancePct`
- `minRewardRiskRatio` must be ≥ 1.0
- `maxDailyLossUsd` must be ≤ `capitalHardLimitUsd × 0.5` (can’t risk losing more than half your cap in a day)
- For HL: `defaultLeverage` ≤ `maxLeverage`, both ≤ 20
- `minConvictionScore` must be between 50–99
- Warnings (not errors): if `maxLeverage > 5`, warn “High leverage increases liquidation risk”; if `maxStopDistancePct > 10`, warn “Wide stops will reduce position sizes significantly”

**`checkSessionActive(userId: string, exchange: string): Promise<{ active: boolean; reason?: string }>`**

- Returns false with reason if: `enabled = false`, current time is outside `activeDays`, current time is past `dailyCutoffTime`, `sessionDurationHours` has elapsed since `sessionStartedAt`, daily trade count from `AutoTradeLog` has hit `maxTradesPerDay`, or daily loss from closed positions today has hit `maxDailyLossUsd`

**`startSession(userId: string, exchange: string): Promise<void>`**

- Sets `sessionStartedAt = now()`, clears `sessionPausedAt`

**`pauseSession(userId: string, exchange: string, reason: string): Promise<void>`**

- Sets `sessionPausedAt = now()`, logs reason to `AutoTradeLog` as action `'SESSION_PAUSED'`

**`getTodayTradeCount(userId: string, exchange: string): Promise<number>`**

- Count `AutoTradeLog` entries for today where `action IN ('BUY', 'SELL')` and `status = 'submitted'`

**`getTodayLossUsd(userId: string, exchange: string): Promise<number>`**

- Sum realized losses from `ClosedPosition` for today, filtered by exchange

## PHASE C — Enforce Parameters in AutoTradeExecutor

In `server/src/services/autotrader/AutoTradeExecutor.ts`, update `evaluateAndExecute()` to load and enforce per-exchange configs:

Before routing any trade to an exchange:

```typescript
const exchangeConfig = await exchangeAutoConfigService.getConfig(userId, exchange);
const sessionCheck = await exchangeAutoConfigService.checkSessionActive(userId, exchange);

if (!sessionCheck.active) {
  logSkip(ticker, `Exchange session inactive: ${sessionCheck.reason}`);
  continue;
}
```

Then enforce each parameter in order:

1. **Capital hard limit**: fetch `PortfolioStateService` → sum notional of all open positions on this exchange. If already at or above `capitalHardLimitUsd`, skip with reason `'CAPITAL_HARD_LIMIT_REACHED'`
1. **Capital target**: if current deployed capital + new position notional would exceed `capitalTargetUsd`, reduce position size to fit, or skip if it would be below `minPositionSizeUsd`
1. **Max position size**: clamp `PositionSizer` output to `maxPositionSizeUsd`. If clamped quantity is below `minPositionSizeUsd`, skip.
1. **Order cooldown**: check `AutoTradeLog` for this exchange — if any order was placed within the last `orderCooldownMinutes` minutes, skip with reason `'ORDER_COOLDOWN'`
1. **Max trades per scan**: count how many trades have been placed in this `evaluateAndExecute()` call for this exchange. Stop if `maxTradesPerScan` reached.
1. **Signal thresholds**: enforce `minConvictionScore`, `minConfidenceScore`, `allowedBias`, `allowedActions` (these may be stricter than the global AutoTradeConfig values — use the MORE restrictive of the two)
1. **Stop distance check**: compute `stopDistancePct = abs(entryPrice - stopLoss) / entryPrice × 100`. Skip if < `minStopDistancePct` or > `maxStopDistancePct`
1. **Reward:risk ratio**: compute `(takeProfit - entryPrice) / (entryPrice - stopLoss)`. Skip if < `minRewardRiskRatio`
1. **HL-specific**: apply `defaultLeverage` (capped at `maxLeverage`); check `allowedAssets` / `blockedAssets`; if `allowShorts = false`, skip any `BEARISH` bias even if `allowedBias` includes it
1. **TOS-specific**: check `allowedAssetTypes`; apply `orderSession` and `orderDuration` to order request; check `minAvgDailyVolume` and `minMarketCapUsd` from the thesis’s market structure data; check `blockedSectors` against the candidate’s sector

## PHASE H — Session Auto-Management

```typescript
// For each exchange with a config:
const hlCheck = await exchangeAutoConfigService.checkSessionActive(userId, 'hyperliquid');
const tosCheck = await exchangeAutoConfigService.checkSessionActive(userId, 'tos');

if (!hlCheck.active && hlConfig.sessionStartedAt && !hlConfig.sessionPausedAt) {
```

Also check `maxDailyLossUsd`: after each monitoring cycle, compute today’s realized P&L on each exchange. If it crosses the `maxDailyLossUsd` threshold (as a loss), call `pauseSession()` with reason `'DAILY_LOSS_LIMIT_REACHED'` and create a critical system alert via the existing `Alert` model.

## IMPLEMENTATION NOTES

- The `ExchangeAutoConfig` record should be created with all defaults on first `GET` if none exists — never return 404 for a missing config
- All dollar values stored in DB as `Float` in USD; display with `$` prefix and 2 decimal places in UI
- The `allowedAssets` and `blockedAssets` tag inputs and `blockedSectors` should store as `String[]` in Prisma using PostgreSQL array type — or as JSON if PostgreSQL arrays aren’t available in the current setup
- `activeDays` stored as `Int[]` (0 = Sunday, 1 = Monday … 6 = Saturday) — the UI checkboxes map to these integers
- When `riskMode` is set to CONSERVATIVE, MODERATE, or AGGRESSIVE, the `customRiskPct` field is ignored even if set — only use it when `riskMode === 'CUSTOM'`
- The preset market cap buttons ($500M/$1B/$5B/$10B) should visually highlight the currently selected tier but the underlying stored value is the raw float
- The tag inputs for `allowedAssets`, `blockedAssets`, and `blockedSectors` should show entered values as small dismissible chips/badges styled with the app’s `bg-surface-3 text-xs rounded` pattern
- Form state should be managed with `useState` initialized from the `useQuery` data; unsaved changes should show a yellow “Unsaved changes” banner near the Save button
- All parameters in these panels are **per-exchange** and **per-user** — two different users on the same system have completely independent configs
- When enforcing `minMarketCapUsd` and `minAvgDailyVolume` in TOS trades, use the market cap and volume data already present in the `ThesisSummary` / `RankedResult` from the scan — do not make additional API calls at execution time
