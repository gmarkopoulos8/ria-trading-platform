Expand the existing RIA BOT Replit project.

Goal:
Add a dedicated Polymarket module that lets users browse Polymarket markets, analyze probabilities and event momentum, track positions in a paper portfolio, and simulate trades using paper money only.

Important implementation rules:
- Do not place live trades
- Do not submit authenticated CLOB orders
- Do not require a private key for the paper-trading workflow
- Use Polymarket public APIs for market discovery and analytics
- Build all buy/sell actions as simulated paper trades stored in the app database

Polymarket integration overview:
Use the official Polymarket public APIs for read-only data:
- Gamma API for market and event metadata
- Data API for positions, analytics, and trade-related public data where applicable
- Optional CLOB public read endpoints for prices, order books, midpoint, spread, and price history

Architecture requirements:
Create a dedicated Polymarket module with:
- /server/services/polymarket/
- /server/routes/polymarket.ts
- /client/src/pages/polymarket/
- /client/src/components/polymarket/

Environment variables:
- POLY_GAMMA_BASE_URL=https://gamma-api.polymarket.com
- POLY_DATA_BASE_URL=https://data-api.polymarket.com
- POLY_CLOB_BASE_URL=https://clob.polymarket.com

Do not require authenticated trading credentials for this phase.

Core features:

1. Polymarket Market Explorer
Build a page where users can:
- browse active markets
- search by keyword
- filter by category, status, event, liquidity, volume, and end date
- view market question, outcomes, implied probabilities, volume, liquidity, end time, and event metadata
- open a detailed market page

2. Market Detail Page
For any selected market, display:
- market question
- event title
- category
- active / closed state
- outcome tokens
- current yes/no prices or midpoint
- spread
- order book snapshot if available
- recent price history
- volume and liquidity stats
- event end date
- market description
- related markets in the same event
- recent momentum summary
- volatility summary
- AI research summary

3. AI Market Thesis for Polymarket
Create a Polymarket Thesis Engine that evaluates each market using:
- recent price movement
- momentum in implied probability
- spread / liquidity quality
- volume trend
- event recency
- market activity
- related-news catalyst analysis
- confidence of current consensus
- reversal risk
- binary event uncertainty

For each market, output:
- Market Health Score from 0 to 100
- Bias: yes, no, or neutral
- Confidence Score
- Liquidity Score
- Risk Score
- Momentum Score
- Thesis Summary
- Supporting Reasons
- Main Risk to Thesis
- Suggested Hold Window
- Action Label:
  - high conviction
  - tradable
  - developing
  - weak
  - avoid

4. Polymarket Paper Trading
Allow the user to simulate opening a paper position on a Polymarket market outcome.

Paper position fields:
- market id
- event id
- question
- selected side: yes or no
- entry probability / price
- quantity
- paper capital allocated
- current mark
- unrealized P&L
- realized P&L
- status
- entry time
- close time
- thesis id
- thesis health
- notes

Users should be able to:
- open simulated yes/no positions
- close simulated positions
- view open paper positions
- view closed paper positions
- compare thesis vs outcome

5. Polymarket Dashboard
Create a dedicated dashboard showing:
