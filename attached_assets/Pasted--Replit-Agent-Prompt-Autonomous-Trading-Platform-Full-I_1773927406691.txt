## CONTEXT

This is an existing full-stack TypeScript trading platform (“RIA BOT”) with:

- **Frontend**: React + Vite + TailwindCSS, React Router v6, TanStack Query
- **Backend**: Node.js + Express + Prisma (PostgreSQL)
- **Exchange integrations**: Hyperliquid (crypto perps) and Schwab/ThinkorSwim (equities/options)
- **AI Analysis**: Multi-agent ThesisEngine (MarketStructure → Catalyst → Risk → Thesis agents)
- **Existing tabs/pages**: Dashboard, Daily Scan, Stock Health, Hyperliquid, Thinkorswim, Opportunity Scanner, Symbol Intelligence, Paper Portfolio, Alert Center, Catalyst Intelligence, Risk Console, Performance Lab, Settings

The platform currently performs analysis and paper trading. The goal is to make it **fully autonomous** — meaning the system scans markets, scores opportunities, decides position sizing, executes live trades on both exchanges, monitors open positions, and auto-exits based on thesis health — all without manual intervention.

**Do NOT remove, rename, or restructure any existing pages, routes, components, or nav items. All existing functionality must continue to work.**

#### 1B. Position Sizer

**File**: `server/src/services/portfolio/PositionSizer.ts`

```typescript
interface SizingInput {
  exchange: 'tos' | 'hyperliquid';
  ticker: string;
  entryPrice: number;
  stopLossPrice: number; // invalidation zone level from ThesisEngine
  convictionScore: number; // 0–100
  riskMode: 'CONSERVATIVE' | 'MODERATE' | 'AGGRESSIVE';
  portfolioState: LivePortfolioState;
}

interface SizingOutput {
  quantity: number; // shares for TOS, contracts/units for HL
  notionalValue: number;
  riskAmount: number; // dollars at risk if stop hit
  riskPct: number; // % of portfolio at risk
  leverageUsed: number; // for HL
  reasoning: string;
}
```

Implement `calculatePositionSize(input: SizingInput): SizingOutput`:

- Base risk per trade: 1% of portfolio equity for CONSERVATIVE, 2% for MODERATE, 3% for AGGRESSIVE
- Scale risk by conviction: multiply base risk by `convictionScore / 100` (so conviction=80 at MODERATE = 1.6% risk)
- `riskAmount = portfolioEquity × riskPct`
- `quantity = riskAmount / (entryPrice - stopLossPrice)` (absolute distance)
- For Hyperliquid: clamp leverage to max 5x, calculate contract size in USD
- Hard limits: never exceed 10% portfolio in a single position; never risk more than 5% in one trade regardless of conviction
- Return quantity = 0 with reasoning if stop is wider than 15% (too risky)
- Round to nearest whole share for TOS; use HL’s `szDecimals` precision for crypto

#### 1C. Auto Trade Executor

**File**: `server/src/services/autotrader/AutoTradeExecutor.ts`

This is the central bridge between scan results and order execution.

```typescript
interface AutoTradeConfig {
  enabled: boolean;
  minConvictionScore: number; // default 78
  maxConcurrentPositions: number; // default 5 total, across both exchanges
  maxPositionsPerExchange: number; // default 3
  riskMode: 'CONSERVATIVE' | 'MODERATE' | 'AGGRESSIVE';
  allowedBias: Array<'BULLISH' | 'BEARISH'>; // default ['BULLISH']
  allowedActions: string[]; // default ['STRONG_BUY', 'BUY']
  stocksEnabled: boolean;
  cryptoEnabled: boolean;
  dryRun: boolean; // mirrors exchange dry run setting
}
```

Implement `evaluateAndExecute(scanRunId: string, config: AutoTradeConfig): Promise<ExecutionSummary>`:

1. Load ranked results from `DailyScanResult` for the given `scanRunId`
1. Filter by: `convictionScore >= config.minConvictionScore`, `bias in config.allowedBias`, `recommendedAction in config.allowedActions`
1. Sort by `compositeScore` descending — best opportunities first
1. Load `portfolioState` from `PortfolioStateService`
1. Count current open positions across both exchanges — skip if already at `maxConcurrentPositions`
1. For each candidate (up to the open slot count):
- Skip if `isAlreadyHolding(ticker)` returns true
- Skip if `hasOpenOrder(ticker)` returns true
- Skip if `getTotalExposurePct(ticker) > 5%`
- Determine exchange: use `hyperliquid` if assetClass is `crypto`, use `tos` if `stock` or `etf`
- Check exchange-specific position count against `maxPositionsPerExchange`
- Extract `entryZone.low` as entry price, `invalidationZone.level` as stop loss, `takeProfit1.level` as target
- Call `PositionSizer.calculatePositionSize()` — skip if quantity = 0
- Place bracket order via `tosExchangeService.placeOrder()` (orderType: ‘BRACKET’, takeProfitPct and stopLossPct derived from levels) or `hyperliquidExchangeService.placeOrder()`
- Log outcome to new `AutoTradeLog` DB table (see Phase 2)
1. Return summary: `{ evaluated: n, skipped: n, placed: n, errors: n, positions: [...] }`

Implement `getExecutorConfig(userId: string): Promise<AutoTradeConfig>` — loads from `UserSettings.autoTradeConfig` JSON field (add field in migration)

Implement `saveExecutorConfig(userId: string, config: Partial<AutoTradeConfig>): Promise<void>`

#### 1D. Intraday Monitoring Loop

**File**: `server/src/services/autotrader/IntradayMonitorLoop.ts`

Implement a loop that runs during market hours and triggers auto-exits:

```typescript
export function startIntradayMonitor(): void
export function stopIntradayMonitor(): void
export function getMonitorStatus(): { running: boolean; lastRun: Date | null; nextRun: Date | null; actionsToday: number }
```

- Poll every 10 minutes during market hours (9:30 AM – 4:00 PM ET weekdays), and every 30 minutes 24/7 for crypto-only positions
- Call `monitorAllOpenPositions()` from existing `PositionMonitor`
- After monitoring, query DB for any `MonitoringAlert` with `isRead = false` and `severity IN ('WARNING', 'CRITICAL')` created in the last 15 minutes
- For each alert matching `alertType IN ('SETUP_INVALIDATED', 'INVALIDATION_THREATENED')`:
  - Check if `autoTradeConfig.enabled` is true for the position’s owner
  - If `alertType === 'SETUP_INVALIDATED'`: call `closePosition()` on the appropriate exchange immediately
  - If `alertType === 'INVALIDATION_THREATENED'` and thesis health < 35: call `closePosition()`
- For `alertType === 'TARGET_APPROACHED'` (thesis health > 80, proximity < 3%):
  - Place a reduce-only limit sell/close at the take-profit level
- Mark acted-upon alerts as read
- Log all auto-actions to `AutoTradeLog`
