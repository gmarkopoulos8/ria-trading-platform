# Replit Agent Prompt — Fix Deployed App (Server Never Starts)

The deployed app at ria-bot.replit.app is broken because main.py is empty (0 bytes). The deployment runs it and immediately exits. The Node server never starts, so login and everything else fails.

---

## FIX 1 — Write main.py

**File**: `main.py` (root)

Replace the empty file:

```python
import subprocess
import sys
import os

if __name__ == "__main__":
    env = os.environ.copy()
    env["NODE_ENV"] = "production"
    result = subprocess.run(
        ["npm", "run", "start"],
        env=env,
        cwd=os.path.dirname(os.path.abspath(__file__)) or "."
    )
    sys.exit(result.returncode)
```

## FIX 3 — Express must serve the React build in production

In dev, Vite runs on port 5000 and proxies /api to port 3001. In production there is no Vite — the Express server must serve the built React files AND handle API routes on the same port.

**File**: `server/src/index.ts`

Add imports near the top:
```typescript
import path from 'path';
import fs   from 'fs';
```

Add this block AFTER all `app.use('/api/...')` registrations, BEFORE `app.use(errorHandler)`:

```typescript
// Serve React build in production
const clientDist = path.resolve(__dirname, '../../client/dist');
if (process.env.NODE_ENV === 'production' && fs.existsSync(clientDist)) {
  app.use(express.static(clientDist));
  app.get('*', (_req, res) => res.sendFile(path.join(clientDist, 'index.html')));
  console.log('   Static      : Serving React build');
}
```


## Required Replit Secrets

Check the lock icon in the Replit sidebar. These MUST be set:

| Key | Value |
|-----|-------|
| `NODE_ENV` | `production` |
| `SESSION_SECRET` | Any random 32+ char string |
| `DATABASE_URL` | Your PostgreSQL connection string |

If `NODE_ENV` is not `production`, the server uses insecure cookies that the browser rejects on HTTPS. If `SESSION_SECRET` is missing, the server exits immediately in prod.
