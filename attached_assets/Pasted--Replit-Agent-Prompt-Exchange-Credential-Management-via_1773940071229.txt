# Replit Agent Prompt — Exchange Credential Management via UI

This is an addendum to all previous prompts. Apply on top of everything already built. Do not remove, rename, or break any existing functionality. The goal is to allow users to connect Hyperliquid and ThinkorSwim directly through the UI instead of manually editing Replit Secrets.

-----

## OVERVIEW

Currently, exchange credentials are read from environment variables (`process.env.HL_WALLET_ADDRESS`, `process.env.SCHWAB_CLIENT_ID`, etc.). This replaces that system with:

1. **Encrypted credential storage in the database** — per-user, AES-256-GCM encrypted
1. **A credential service** that loads from DB first, falls back to env vars (for backward compatibility)
1. **Updated config objects** (`HL_CONFIG`, `TOS_CONFIG`) that read from the credential service instead of `process.env` directly
1. **Connection forms** on the Hyperliquid and ThinkorSwim dashboard pages
1. **Status cards on the main Dashboard** showing each exchange’s connection state and live balance

The only Replit Secret that must be set manually is `ENCRYPTION_KEY` — a 32-byte hex string used to encrypt/decrypt all credentials. Everything else goes through the UI.

## PHASE 1 — Encryption Utility

**File**: `server/src/lib/encryption.ts`

```typescript
import crypto from 'crypto';

const ALGORITHM = 'aes-256-gcm';
const KEY_LENGTH = 32; // bytes
const IV_LENGTH = 12;  // bytes (96-bit IV for GCM)
const TAG_LENGTH = 16; // bytes

function getKey(): Buffer {
  const keyHex = process.env.ENCRYPTION_KEY ?? '';
  if (!keyHex || keyHex.length < 64) {
    throw new Error('ENCRYPTION_KEY must be set in Replit Secrets. Generate with: node -e "console.log(require(\'crypto\').randomBytes(32).toString(\'hex\'))"');
  }
  return Buffer.from(keyHex.slice(0, 64), 'hex');
}

export function encrypt(plaintext: string): string {
  const key = getKey();
  const iv = crypto.randomBytes(IV_LENGTH);
  const cipher = crypto.createCipheriv(ALGORITHM, key, iv);
  
  const encrypted = Buffer.concat([
    cipher.update(plaintext, 'utf8'),
    cipher.final(),
  ]);
  const tag = cipher.getAuthTag();
  
  // Format: iv(hex):tag(hex):ciphertext(hex)
  return `${iv.toString('hex')}:${tag.toString('hex')}:${encrypted.toString('hex')}`;
}

export function decrypt(ciphertext: string): string {
  const key = getKey();
  const parts = ciphertext.split(':');
  if (parts.length !== 3) throw new Error('Invalid encrypted format');
  
  const [ivHex, tagHex, encryptedHex] = parts;
  const iv = Buffer.from(ivHex, 'hex');
  const tag = Buffer.from(tagHex, 'hex');
  const encrypted = Buffer.from(encryptedHex, 'hex');
  
  const decipher = crypto.createDecipheriv(ALGORITHM, key, iv);
  decipher.setAuthTag(tag);
  
  return Buffer.concat([
    decipher.update(encrypted),
    decipher.final(),
  ]).toString('utf8');
}

export function encryptIfPresent(value: string | null | undefined): string | null {
  if (!value || value.trim() === '') return null;
  return encrypt(value.trim());
}

export function decryptIfPresent(value: string | null | undefined): string | null {
  if (!value) return null;
  try {
    return decrypt(value);
  } catch {
    return null;
  }
}

// Validate that ENCRYPTION_KEY is set and valid without throwing
export function isEncryptionConfigured(): boolean {
  try {
    getKey();
    return true;
  } catch {
    return false;
  }
}
```

## PHASE 2 — Database Schema

Add to `server/prisma/schema.prisma`:

```prisma
model ExchangeCredential {
  id        String   @id @default(cuid())
  userId    String
  exchange  String   // 'hyperliquid' | 'tos'
  
  // All sensitive values stored AES-256-GCM encrypted
  // Hyperliquid fields

  accountNumber     String?  // stored plaintext (not sensitive alone)
  
  // Shared settings
  dryRun            Boolean  @default(true)
  maxDrawdownPct    Float    @default(5.0)
  defaultLeverage   Int      @default(2)   // HL only
  
  // Connection state
  isConnected       Boolean  @default(false)
  lastVerifiedAt    DateTime?
  lastError         String?
  
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  @@unique([userId, exchange])
  @@index([userId])
}
```

## PHASE 3 — Credential Service

**File**: `server/src/services/credentials/CredentialService.ts`

This service is the single source of truth for all exchange credentials. It loads from DB first, falls back to env vars.

```typescript
import { prisma } from '../../lib/prisma';
import { encrypt, decrypt, encryptIfPresent, decryptIfPresent } from '../../lib/encryption';

export interface HLCredentials {
  walletAddress: string;
  agentPrivateKey: string | null;
  isMainnet: boolean;
  dryRun: boolean;
  maxDrawdownPct: number;
  defaultLeverage: number;
  source: 'database' | 'env';
}

export interface TOSCredentials {
  clientId: string;
  clientSecret: string;
  redirectUri: string;
  refreshToken: string;
  accountNumber: string;
  dryRun: boolean;
  maxDrawdownPct: number;
  source: 'database' | 'env';
}
```

**`getHLCredentials(userId: string): Promise<HLCredentials | null>`**:

1. Try DB: `prisma.exchangeCredential.findUnique({ where: { userId_exchange: { userId, exchange: 'hyperliquid' } } })`
1. If found and `walletAddress` is set: decrypt `encryptedAgentKey`, return as `HLCredentials` with `source: 'database'`
1. If not in DB: fall back to env vars (`process.env.HL_WALLET_ADDRESS`, etc.) with `source: 'env'`
1. Return null if neither source has credentials

**`getTOSCredentials(userId: string): Promise<TOSCredentials | null>`**:

- Same pattern: DB first, env vars fallback
- Decrypt `encryptedSecret` and `encryptedRefreshToken` when loading from DB

**`saveHLCredentials(userId: string, creds: { walletAddress: string; agentPrivateKey: string; isMainnet: boolean; dryRun?: boolean; maxDrawdownPct?: number; defaultLeverage?: number }): Promise<void>`**:

- Encrypt `agentPrivateKey` before storing
- Upsert into `ExchangeCredential`
- Set `isConnected: false` — connection must be verified separately

**`saveTOSCredentials(userId: string, creds: { clientId: string; clientSecret: string; redirectUri: string; refreshToken: string; accountNumber: string; dryRun?: boolean }): Promise<void>`**:

- Encrypt `clientSecret` and `refreshToken` before storing
- Upsert into `ExchangeCredential`

**`updateTOSRefreshToken(userId: string, newRefreshToken: string): Promise<void>`**:

- Called when Schwab returns a new refresh token during token refresh
- Encrypts and updates `encryptedRefreshToken` in DB

**`markConnected(userId: string, exchange: string, error?: string): Promise<void>`**:

- Updates `isConnected: !error`, `lastVerifiedAt: new Date()`, `lastError: error ?? null`

**`deleteCredentials(userId: string, exchange: string): Promise<void>`**:

- Deletes the credential record — disconnects the exchange

**`getConnectionStatus(userId: string): Promise<{ hl: ConnectionStatus; tos: ConnectionStatus }>`**:

```typescript
interface ConnectionStatus {
  isConnected: boolean;
  lastVerifiedAt: Date | null;
  lastError: string | null;
  source: 'database' | 'env' | 'none';
  dryRun: boolean;
  // Exchange-specific:
  walletAddress?: string;        // HL: last 6 chars only
  accountNumber?: string;        // TOS: last 4 digits only
  isMainnet?: boolean;           // HL only
}
```

## PHASE 4 — Update Config Objects to Use Credential Service

The existing `HL_CONFIG` and `TOS_CONFIG` objects read from `process.env`. Update them to support a runtime credential override from the DB. The cleanest approach: keep the env var fallback exactly as-is, and add a credential injection layer.

### 4C. Credential Loader at Startup

**File**: `server/src/services/credentials/CredentialLoader.ts`

```typescript
import { credentialService } from './CredentialService';
import { setHLRuntimeCredentials } from '../hyperliquid/hyperliquidConfig';
import { setTOSRuntimeCredentials } from '../tos/tosConfig';

// For single-user mode: load the first user's credentials at startup
// For multi-user mode later: call loadUserCredentials(userId) per-request
export async function loadDefaultCredentials(): Promise<void> {
  try {
    const firstUser = await prisma.user.findFirst({ orderBy: { createdAt: 'asc' } });
    if (!firstUser) return;
    
    const hlCreds = await credentialService.getHLCredentials(firstUser.id);
    if (hlCreds && hlCreds.source === 'database') {
      setHLRuntimeCredentials({
        walletAddress: hlCreds.walletAddress,
        agentPrivateKey: hlCreds.agentPrivateKey ?? undefined,
        isMainnet: hlCreds.isMainnet,
        dryRun: hlCreds.dryRun,
        maxDrawdownPct: hlCreds.maxDrawdownPct,
        defaultLeverage: hlCreds.defaultLeverage,
      });
    }
    
    const tosCreds = await credentialService.getTOSCredentials(firstUser.id);
    if (tosCreds && tosCreds.source === 'database') {
      setTOSRuntimeCredentials({
        clientId: tosCreds.clientId,
        clientSecret: tosCreds.clientSecret,
        redirectUri: tosCreds.redirectUri,
        refreshToken: tosCreds.refreshToken,
        accountNumber: tosCreds.accountNumber,
        dryRun: tosCreds.dryRun,
        maxDrawdownPct: tosCreds.maxDrawdownPct,
      });
    }
    
    console.info('[CredentialLoader] Exchange credentials loaded from database');
  } catch (err) {
    console.warn('[CredentialLoader] Could not load credentials from DB:', (err as Error).message);
  }
}
```

Also update `tosAuthService.ts` in the `refreshAccessToken()` function: when a new refresh token is returned by Schwab, also call `credentialService.updateTOSRefreshToken(userId, newRefreshToken)` to keep the DB in sync. The `userId` comes from the session on the request that triggered the refresh — pass it through as a parameter or use the first user ID in single-user mode.

## PHASE 5 — API Routes for Credential Management

**File**: `server/src/routes/credentials.ts`

```
POST /api/credentials/hl/connect         → save + verify HL credentials
POST /api/credentials/hl/disconnect      → delete HL credentials
GET  /api/credentials/hl/status          → connection status (no secrets returned)
POST /api/credentials/tos/connect        → save + verify TOS credentials (step 2: code exchange)
POST /api/credentials/tos/auth-url       → generate Schwab OAuth URL (step 1)
POST /api/credentials/tos/disconnect     → delete TOS credentials
GET  /api/credentials/tos/status         → connection status
GET  /api/credentials/status             → both exchanges in one call (used by Dashboard)
PATCH /api/credentials/hl/settings       → update dryRun, maxDrawdownPct, leverage only
PATCH /api/credentials/tos/settings      → update dryRun, maxDrawdownPct only
```

### Route Implementations:

**`POST /api/credentials/hl/connect`**:

```typescript
// Body: { walletAddress, agentPrivateKey, isMainnet, dryRun?, maxDrawdownPct?, defaultLeverage? }
router.post('/hl/connect', requireAuth, async (req, res) => {
  const { walletAddress, agentPrivateKey, isMainnet = true, dryRun = true, maxDrawdownPct = 8, defaultLeverage = 2 } = req.body;
  
  // Validate inputs
  if (!walletAddress?.startsWith('0x') || walletAddress.length !== 42) {
    return res.status(400).json({ success: false, error: 'Invalid wallet address. Must be a 42-character address starting with 0x.' });
  }
  if (!agentPrivateKey?.startsWith('0x') || agentPrivateKey.length !== 66) {
    return res.status(400).json({ success: false, error: 'Invalid private key. Must be 66 characters starting with 0x.' });
  }
  
  // Verify connection BEFORE saving — call Hyperliquid API with these credentials
  try {
    // Temporarily set runtime creds for verification
    setHLRuntimeCredentials({ walletAddress, agentPrivateKey, isMainnet, dryRun: true }); // always dryRun=true during verify
    
    // Test: fetch user state from Hyperliquid
    const userState = await getUserState(); // from hyperliquidInfoService
    const accountValue = parseFloat(userState?.marginSummary?.accountValue ?? '0');
    
    // Save to DB
    await credentialService.saveHLCredentials(req.session.userId!, {
      walletAddress, agentPrivateKey, isMainnet, dryRun, maxDrawdownPct, defaultLeverage
    });
    
    // Reload runtime creds with the saved settings (including user's dryRun preference)
    await loadDefaultCredentials();
    
    await credentialService.markConnected(req.session.userId!, 'hyperliquid');
    
    return res.json({
      success: true,
      data: {
        message: 'Hyperliquid connected successfully',
        accountValue,
        walletAddress: walletAddress.slice(0, 6) + '...' + walletAddress.slice(-4),
        isMainnet,
        dryRun,
      }
    });
  } catch (err) {
    // Clear the temp runtime creds if verification failed
    clearHLRuntimeCredentials();
    await loadDefaultCredentials(); // restore previous state
    
    const msg = err instanceof Error ? err.message : 'Connection failed';
    await credentialService.markConnected(req.session.userId!, 'hyperliquid', msg);
    return res.status(400).json({ success: false, error: `Connection failed: ${msg}` });
  }
});
```

**`POST /api/credentials/tos/auth-url`** (Step 1 of TOS OAuth):

```typescript
// Body: { clientId, clientSecret, redirectUri }
router.post('/tos/auth-url', requireAuth, async (req, res) => {
  const { clientId, clientSecret, redirectUri = 'https://127.0.0.1' } = req.body;
  
  if (!clientId || !clientSecret) {
    return res.status(400).json({ success: false, error: 'App Key and App Secret are required' });
  }
  
  // Temporarily store clientId/secret in session for the callback step
  req.session.pendingTosAuth = { clientId, clientSecret, redirectUri };
  
  // Build the Schwab OAuth URL
  const params = new URLSearchParams({
    response_type: 'code',
    client_id: clientId,
    redirect_uri: redirectUri,
    scope: 'readonly trader',
  });
  const authUrl = `https://api.schwabapi.com/v1/oauth/authorize?${params.toString()}`;
  
  return res.json({ success: true, data: { authUrl } });
});
```

**`POST /api/credentials/tos/connect`** (Step 2: exchange code for tokens):

```typescript
// Body: { authorizationCode, accountNumber }
router.post('/tos/connect', requireAuth, async (req, res) => {
  const { authorizationCode, accountNumber } = req.body;
  const pending = req.session.pendingTosAuth;
  
  if (!pending) {
    return res.status(400).json({ success: false, error: 'Start the OAuth flow first via /tos/auth-url' });
  }
  if (!authorizationCode) {
    return res.status(400).json({ success: false, error: 'Authorization code is required' });
  }
  if (!accountNumber) {
    return res.status(400).json({ success: false, error: 'Account number is required' });
  }
  
  try {
    // Temporarily set runtime creds for the token exchange
    setTOSRuntimeCredentials({
      clientId: pending.clientId,
      clientSecret: pending.clientSecret,
      redirectUri: pending.redirectUri,
      refreshToken: '',  // not yet available
      accountNumber,
      dryRun: true,
    });
    
    // Exchange code for tokens using existing exchangeCodeForTokens()
    const tokens = await exchangeCodeForTokens(authorizationCode);
    
    // Verify account access
    const account = await getPrimaryAccount();
    const equity = account?.securitiesAccount?.currentBalances?.equity ?? 0;
    
    // Save everything to DB
    await credentialService.saveTOSCredentials(req.session.userId!, {
      clientId: pending.clientId,
      clientSecret: pending.clientSecret,
      redirectUri: pending.redirectUri,
      refreshToken: tokens.refreshToken,
      accountNumber,
      dryRun: true, // always start in dry run
    });
    
    delete req.session.pendingTosAuth;
    
    await loadDefaultCredentials();
    await credentialService.markConnected(req.session.userId!, 'tos');
    
    return res.json({
      success: true,
      data: {
        message: 'ThinkorSwim connected successfully',
        accountNumber: '...' + accountNumber.slice(-4),
        equity,
        dryRun: true,
      }
    });
  } catch (err) {
    clearTOSRuntimeCredentials();
    await loadDefaultCredentials();
    const msg = err instanceof Error ? err.message : 'Connection failed';
    await credentialService.markConnected(req.session.userId!, 'tos', msg);
    return res.status(400).json({ success: false, error: `Connection failed: ${msg}` });
  }
});
```

function TOSConnectCard({ onConnected }: { onConnected: () => void }) {
  const [step, setStep] = useState<'credentials' | 'authorize' | 'callback'>('credentials');
  const [clientId, setClientId] = useState('');
  const [clientSecret, setClientSecret] = useState('');
  const [redirectUri, setRedirectUri] = useState('https://127.0.0.1');
  const [authUrl, setAuthUrl] = useState('');
  const [authCode, setAuthCode] = useState('');
  const [accountNumber, setAccountNumber] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');
  const [showSecret, setShowSecret] = useState(false);

  // Step 1: Generate auth URL
  const handleGenerateUrl = async () => {
    setError('');
    setIsLoading(true);
    try {
      const result = await api.credentials.tosAuthUrl({ clientId, clientSecret, redirectUri });
      setAuthUrl(result.data?.data?.authUrl);
      setStep('authorize');
    } catch (err: any) {
      setError(err?.response?.data?.error ?? 'Failed to generate authorization URL');
    } finally {
      setIsLoading(false);
    }

    setError('');
    setIsLoading(true);

      await api.credentials.tosConnect({ authorizationCode: authCode, accountNumber });

      onConnected();
    } catch (err: any) {
      setError(err?.response?.data?.error ?? 'Connection failed. Check your authorization code.');
    } finally {
      setIsLoading(false);
    }

      <Card className="w-full max-w-md p-6">

          ))}
        </div>

        {step === 'credentials' && (
          <div className="space-y-4">

              <label className="text-xs text-slate-500 font-mono uppercase tracking-wider mb-1.5 block">App Key (Client ID)</label>
              <input type="text" value={clientId} onChange={(e) => setClientId(e.target.value)}
                placeholder="Your Schwab app key"
                className="w-full bg-surface-3 border border-surface-border rounded-lg px-3 py-2.5 text-sm font-mono text-white placeholder-slate-600 focus:outline-none focus:border-accent-purple" />

                placeholder="https://127.0.0.1/?code=..."
