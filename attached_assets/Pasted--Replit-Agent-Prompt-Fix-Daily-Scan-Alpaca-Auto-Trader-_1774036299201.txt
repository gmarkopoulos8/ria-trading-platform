# Replit Agent Prompt — Fix Daily Scan + Alpaca Auto Trader

This fixes all the bugs causing the daily scan to get stuck on "Initializing" and the Alpaca auto trader "Start Trading" to fail. Apply every change exactly as written.

## BUG 1 — Daily Scan Blocks HTTP Response (Times Out)

The `/trigger` route does `await runDailyScan()` which runs the entire 2–10 minute scan inside one HTTP request. Replit kills HTTP connections after ~30 seconds. The scan ID is never returned, the UI has nothing to connect to, and it sits on "Initializing" forever.

**File**: `server/src/routes/daily-scans.ts`

Replace the entire `/trigger` route:

```typescript
router.post('/trigger', async (req: Request, res: Response) => {
  try {
    const {
      runType        = 'MANUAL',
      marketSession  = 'MARKET_OPEN',
      assetScope     = 'ALL',
      riskMode       = 'ALL',
      force          = false,
      fullUniverse   = false,
      filterCriteria = {},
    } = req.body ?? {};

    // Check duplicate before creating anything
    if (!(force === true || force === 'true')) {
      const { hasDuplicateRunToday } = await import('../services/scans/scanPersistenceService');
      const isDuplicate = await hasDuplicateRunToday(runType, marketSession);
      if (isDuplicate) {
        return res.status(400).json({
          success: false,
          error: 'A scan for this session already ran today. Enable Force to override.',
        });
      }
    }

    // Pre-create the DB record so we have an ID to return RIGHT NOW
    const { createScanRun, markRunStarted } = await import('../services/scans/scanPersistenceService');
    const scanRun = await createScanRun({
      runType,
      marketSession,
      assetScope:         assetScope as any,
      riskMode:           riskMode as any,
      isFullUniverseScan: fullUniverse,
    });
    await markRunStarted(scanRun.id);

    // Return immediately — UI can now connect SSE stream
    res.json({ success: true, data: { scanRunId: scanRun.id } });

    // Run the actual scan in the background (non-blocking)
    const { runDailyScan } = await import('../services/scans/dailyScanOrchestrator');
    runDailyScan({
      runType,
      marketSession,
      assetScope:          assetScope as any,
      riskMode:            riskMode as any,
      skipDuplicateCheck:  true,
      fullUniverse,
      filterCriteria,
      _existingScanRunId:  scanRun.id,
    }).catch((err: any) => {
      console.error('[Trigger] Background scan failed:', err?.message);
    });

  } catch (err) {
    res.status(400).json({ success: false, error: err instanceof Error ? err.message : 'Scan trigger failed' });
  }
});
```

Also add the prisma import at the top of `daily-scans.ts` if not already there:
```typescript
import { prisma } from '../lib/prisma';
```

And add a reset-stuck route before `export default router`:
```typescript
router.post('/reset-stuck', async (req: Request, res: Response) => {
  try {
    const result = await prisma.dailyScanRun.updateMany({
      where: { status: 'RUNNING' },
      data:  { status: 'FAILED', errorMessage: 'Manually reset via API' },
    });
    res.json({ success: true, data: { reset: result.count } });
  } catch (err) {
    res.status(500).json({ success: false, error: err instanceof Error ? err.message : 'Reset failed' });
  }
});
```

## BUG 3 — Stuck RUNNING Scans Block All New Runs

If a previous scan crashed or the server restarted mid-scan, it leaves a `RUNNING` record in the DB. The duplicate check blocks all new scans. Fix by resetting these on startup.

**File**: `server/src/index.ts`

Add this import near the top:
```typescript
import { prisma } from './lib/prisma';
```

Add this function and call it inside `app.listen`:
```typescript
async function resetStuckScans() {
  try {
    const result = await prisma.dailyScanRun.updateMany({
      where: { status: 'RUNNING' },
      data:  { status: 'FAILED', errorMessage: 'Reset on server startup — was stuck in RUNNING state' },
    });
    if (result.count > 0) {
      console.log(`[Startup] Reset ${result.count} stuck RUNNING scan(s)`);
    }
  } catch (err: any) {
    console.warn('[Startup] Could not reset stuck scans:', err?.message);
  }
}
```

Inside `app.listen`, call it before `startDailyScanScheduler()`:
```typescript
app.listen(PORT, async () => {
  console.log(`✅ RIA BOT API running on port ${PORT}`);
  // ... existing log lines ...

  await resetStuckScans();      // ADD THIS LINE
  await loadDefaultCredentials();
  startDailyScanScheduler();
  // ... rest unchanged
});
```

## BUG 4 — Alpaca Entry Price Uses TOS (Fails When TOS Not Connected)

`resolveEntryPrice()` calls `tosInfoService.getQuotes()` for all stocks including PAPER exchange trades. If TOS isn't connected it returns 0 and the trade is rejected with "Could not resolve entry price."

**File**: `server/src/services/autotrader/AutoTradeExecutor.ts`

Replace the entire `resolveEntryPrice` function:

```typescript
async function resolveEntryPrice(signal: AutoTradeSignal): Promise<number> {
  if (signal.entryPrice && signal.entryPrice > 0) return signal.entryPrice;

  const exchange = signal.exchange ?? 'PAPER';

  try {
    if (signal.assetClass === 'crypto' || signal.assetClass === 'CRYPTO') {
      const { getAssetPrice } = await import('../hyperliquid/hyperliquidInfoService');
      const price = await getAssetPrice(signal.symbol);
      if (price && price > 0) return price;
    }

    // For PAPER: use Yahoo Finance directly — never depends on TOS being connected
    if (exchange === 'PAPER') {
      try {
        const { stocksService } = await import('../market/stocks/StocksService');
        const quote = await stocksService.quote(signal.symbol);
        if (quote?.price && quote.price > 0) return quote.price;
      } catch { /* fall through */ }
    }

    // For TOS exchange only
    if (exchange === 'TOS') {
      try {
        const { getQuotes } = await import('../tos/tosInfoService');
        const quotes = await getQuotes([signal.symbol]);
        const q = quotes[signal.symbol];
        if (q) return q.lastPrice ?? q.mark ?? q.askPrice ?? 0;
      } catch { /* fall through */ }
    }

    // Universal fallback — Yahoo Finance
    const { stocksService } = await import('../market/stocks/StocksService');
    const quote = await stocksService.quote(signal.symbol);
    if (quote?.price && quote.price > 0) return quote.price;

  } catch { /* ignore */ }

  return 0;
}
```

**File**: `client/src/pages/DailyScan.tsx`

Add a mutation near the other mutations:
```typescript
const resetStuckMut = useMutation({
  mutationFn: () => (api.scans as any).resetStuck(),
  onSuccess: () => {
    toast.success('Stuck scan reset — you can now run a new scan');
    qc.invalidateQueries({ queryKey: ['daily-scan-runs'] });
    qc.invalidateQueries({ queryKey: ['daily-scan-latest'] });
  },
  onError: (e: any) => toast.error(e?.response?.data?.error ?? 'Reset failed'),
});
```
