# Replit Agent Prompt — Alpaca Paper Trading Tab

This is an addendum to all previous prompts. Apply on top of everything already built. Do not remove, rename, or break any existing functionality.

-----

## OVERVIEW

Add a full **“Alpaca Paper”** tab to the platform that integrates with Alpaca’s paper trading API at `https://paper-api.alpaca.markets`. This tab mirrors the ThinkorSwim tab in structure and follows all the same patterns (connection card, three-level controls, killswitch, drawdown monitor, order placement) but is purpose-built for testing. It adds five testing-specific features not present on any other tab:

1. **Test Suite** — automated sequence of test orders that verifies all order types work end-to-end
1. **Strategy Replay** — re-run a past daily scan through the auto trader against Alpaca paper
1. **Fill Simulator panel** — shows submitted price vs actual fill price for every order (slippage tracking)
1. **Latency Monitor** — tracks order submission → fill confirmation time in milliseconds
1. **Paper vs Live Comparison** — split view of Alpaca paper positions alongside live TOS/HL positions

The Alpaca paper URL is hardcoded as `https://paper-api.alpaca.markets` — this tab **never** connects to `https://api.alpaca.markets` (live). This is enforced at the service level, not just the UI.

-----

## ALPACA API REFERENCE

**Base URL (hardcoded, never changes)**: `https://paper-api.alpaca.markets`
**Market Data URL**: `https://data.alpaca.markets`

**Authentication**: Every request sends two headers:

```
APCA-API-KEY-ID: {apiKeyId}
APCA-API-SECRET-KEY: {secretKey}
```

**Key endpoints used:**

```
GET  /v2/account                          → account info, equity, buying power, P&L
GET  /v2/positions                        → all open positions
DELETE /v2/positions                      → close all positions at market
DELETE /v2/positions/{symbol}             → close a specific position
GET  /v2/orders?status=open               → open orders
GET  /v2/orders?status=all&limit=50       → order history
POST /v2/orders                           → place order
DELETE /v2/orders/{order_id}              → cancel order
DELETE /v2/orders                         → cancel all orders
GET  /v2/account/portfolio/history        → portfolio value over time (for equity curve)
GET  /v2/clock                            → market clock (is_open, next_open, next_close)
GET  /v2/assets/{symbol}                  → asset info (tradable, fractionable, shortable)
```

**Order placement body** (`POST /v2/orders`):

```json
{
  "symbol": "AAPL",
  "qty": "1",               // or "notional": "100" for dollar-based
  "side": "buy",            // "buy" | "sell"
  "type": "market",         // "market" | "limit" | "stop" | "stop_limit" | "trailing_stop"
  "time_in_force": "day",   // "day" | "gtc" | "ioc" | "fok"
  "limit_price": "150.00",  // required for limit/stop_limit
  "stop_price": "148.00",   // required for stop/stop_limit
  "order_class": "bracket", // "" | "bracket" | "oco" | "oto"
  "take_profit": { "limit_price": "160.00" },  // for bracket orders
  "stop_loss": { "stop_price": "145.00" },     // for bracket orders
  "extended_hours": false,
  "client_order_id": "riabot-{uuid}"  // always set this for tracking
}
```

**Account response fields used:**

- `equity` — total account value
- `buying_power` — available buying power
- `cash` — cash balance
- `last_equity` — equity at last market close (for daily P&L)
- `daytrade_count` — number of day trades in last 5 days
- `pattern_day_trader` — PDT flag
- `trading_blocked` — account restricted

**Position response fields used:**

- `symbol`, `qty`, `side` (`long`/`short`)
- `avg_entry_price` — average entry price
- `current_price` — current market price
- `unrealized_pl` — unrealized P&L in dollars
- `unrealized_plpc` — unrealized P&L as decimal (multiply by 100 for %)
- `market_value` — current position value

**Order response fields used:**

- `id` — Alpaca order ID
- `client_order_id` — our tracking ID
- `symbol`, `qty`, `side`, `type`, `status`
- `limit_price`, `stop_price`
- `filled_qty`, `filled_avg_price`
- `submitted_at`, `filled_at` — timestamps for latency tracking
- `legs` — child orders for bracket orders

```typescript
import axios from 'axios';
import { ALPACA_PAPER_URL, getAlpacaCredentials } from './alpacaConfig';

function authHeaders() {
  const creds = getAlpacaCredentials();
  if (!creds) throw new Error('Alpaca credentials not configured');
  return {
    'APCA-API-KEY-ID': creds.apiKeyId,
    'APCA-API-SECRET-KEY': creds.secretKey,
  };
}

async function alpacaGet<T>(path: string, params?: Record<string, unknown>): Promise<T> {
  const { data } = await axios.get<T>(`${ALPACA_PAPER_URL}${path}`, {
    headers: authHeaders(),
    params,
    timeout: 10_000,
  });
  return data;
}
```

## PHASE 5 — Testing Features (Backend)

### 5A. Test Suite Service

**File**: `server/src/services/alpaca/AlpacaTestSuite.ts`

```typescript
export interface TestCase {
  name: string;
  description: string;
  status: 'pending' | 'running' | 'passed' | 'failed' | 'skipped';
  durationMs?: number;
  error?: string;
  result?: unknown;
}

export interface TestSuiteResult {
  runId: string;
  startedAt: Date;
  completedAt?: Date;
  passed: number;
  failed: number;
  skipped: number;
  tests: TestCase[];
}
```

**`runTestSuite(userId: string): Promise<TestSuiteResult>`**

Runs these tests in sequence, each as its own `TestCase`. All orders use `qty: 1` for minimal capital usage. Wait up to 10 seconds for each fill before marking as timed out:

1. **“Market Buy”** — Buy 1 share of SPY at market. Expect: order status transitions to `filled`.
1. **“Market Sell”** — Sell the 1 share of SPY from test 1 at market. Expect: filled, position closed.
1. **“Limit Order Place”** — Place a limit buy of AAPL 10% below current price (won’t fill). Expect: order status `new` or `pending_new`.
1. **“Limit Order Cancel”** — Cancel the order from test 3. Expect: status becomes `canceled`.
1. **“Bracket Order”** — Buy 1 share of MSFT at market with bracket: take profit +5%, stop loss -3%. Expect: parent order filled, two child orders (limit sell + stop sell) appear as legs.
1. **“Bracket Order Close”** — Cancel the bracket and close the MSFT position. Expect: position flat.
1. **“Fractional Order”** — Buy $10 notional of AAPL (fractional). Expect: filled with fractional qty.
1. **“Account Balance Check”** — Verify equity > 0 and buying_power > 0. Expect: both values positive.
1. **“Position Count Verification”** — After tests 1-7 close all positions, verify position count is 0. Expect: empty positions array.
1. **“Latency Check”** — Place and immediately cancel a market order. Measure round-trip time. Expect: < 5000ms. Record actual latency.

For each test: record `durationMs` as the time from start to completion. If a test throws, catch the error, mark as `failed`, store the error message, and continue to the next test. Never abort the suite on a single failure.

Add route: `POST /api/alpaca/test-suite/run` → runs the suite, returns `TestSuiteResult`
Add route: `GET /api/alpaca/test-suite/last` → returns the most recent result from a module-level variable

### 5B. Strategy Replay Service

**File**: `server/src/services/alpaca/StrategyReplayService.ts`

```typescript
export interface ReplayOptions {
  scanRunId: string;          // which past scan to replay
  userId: string;
  maxPositions?: number;      // default 3
  minConviction?: number;     // default 75
  capitalPerTrade?: number;   // default 500 (USD notional)
  useCurrentPrices?: boolean; // true = use today's prices, false = use prices from scan date
}

export interface ReplayResult {
  replayId: string;
  scanRunId: string;
  scanDate: Date;
  candidatesEvaluated: number;
  ordersPlaced: number;
  ordersSkipped: number;
  skippedReasons: Array<{ symbol: string; reason: string }>;
  placedOrders: Array<{ symbol: string; orderId: string; price: number; conviction: number }>;
  startedAt: Date;
}
```

**`runStrategyReplay(opts: ReplayOptions): Promise<ReplayResult>`**:

1. Load the `DailyScanRun` and its `DailyScanResult` records for `opts.scanRunId`
1. Verify the scan is completed — error if not
1. Filter results: `convictionScore >= minConviction`, `bias === 'BULLISH'`, `recommendedAction IN ['STRONG_BUY', 'BUY']`
1. Sort by `compositeScore` descending
1. Take top `maxPositions` candidates
1. For each candidate:
- Check Alpaca account has enough buying power
- Verify asset is tradable via `getAsset(symbol)` — skip if not tradable or not fractionable when using notional
- Place a notional buy order via `alpacaExchangeService.placeOrder()` with `notional: opts.capitalPerTrade`, `type: 'market'`
- Tag the `AlpacaOrderLog` with `scanRunId` for tracking
1. Return the full `ReplayResult`

Add route: `POST /api/alpaca/replay` → `{ scanRunId, options... }`
Add route: `GET /api/alpaca/replay/history` → list past replays from DB (query `AlpacaOrderLog` where `scanRunId IS NOT NULL`, group by `scanRunId`)

### 5C. Latency Monitor

**File**: `server/src/services/alpaca/LatencyMonitor.ts`

A polling service that checks recently submitted orders and measures time to fill:

```typescript
interface LatencyRecord {
  orderId: string;
  symbol: string;
  submittedAt: Date;
  filledAt: Date | null;
  latencyMs: number | null;
  status: string;
}
```

**`startLatencyMonitor(intervalMs = 5000): void`**

- Every 5 seconds: fetch all orders from `AlpacaOrderLog` where `status = 'submitted'` and `filledAt IS NULL` and `submittedAt > now - 5 minutes`
- For each, call `GET /v2/orders/{alpacaOrderId}` to check current status
- If `status === 'filled'` and `filled_at` is present:
  - Compute `latencyMs = new Date(filled_at).getTime() - submittedAt.getTime()`
  - Update `AlpacaOrderLog`: set `filledAt`, `latencyMs`, `filledPrice = filled_avg_price`
  - Compute `slippagePct = ((filledPrice - submittedPrice) / submittedPrice) * 100`
  - Update `slippagePct` in DB

**`getLatencyStats(userId: string): Promise<LatencyStats>`**:

```typescript
interface LatencyStats {
  avgLatencyMs: number;
  minLatencyMs: number;
  maxLatencyMs: number;
  p95LatencyMs: number;
  totalOrdersTracked: number;
  avgSlippagePct: number;
  lastUpdated: Date;
}
```

Queries `AlpacaOrderLog` where `latencyMs IS NOT NULL` and `userId = userId`.

Add route: `GET /api/alpaca/latency/stats`

## IMPORTANT NOTES

**Never use live Alpaca URL.** The string `https://api.alpaca.markets` must never appear anywhere in the codebase. Only `https://paper-api.alpaca.markets`. Add a comment in `alpacaConfig.ts`: `// HARDCODED PAPER URL — do NOT change to live`.

**Dry Run on Alpaca paper** still means something: when `dryRun: true`, order requests are logged to `AlpacaOrderLog` with `isDryRun: true` but not sent to Alpaca. This lets you see what the auto trader would do without even touching your paper balance. Default is `dryRun: true`.

**The Test Suite should only run in paper mode.** Add a guard in `runTestSuite()` that throws if somehow called with live credentials (it won’t happen with hardcoded URL, but belt-and-suspenders).

**Strategy Replay** operates independently from the Auto Trader’s normal scan cycle. It only affects the Alpaca paper account — it cannot place orders on TOS or Hyperliquid.

**Latency numbers in paper trading** reflect Alpaca’s simulation speed, not real exchange latency. Add a tooltip on the latency card explaining this distinction.

**`client_order_id` prefix** should always be `riabot-` to identify orders placed by this platform in Alpaca’s dashboard.
