# Replit Agent Prompt — Three-Level Trading Controls

This is an addendum to all previous prompts. Apply on top of everything already built. Do not remove or rename any existing functions, routes, or UI elements — only extend them.

-----

## OVERVIEW

The existing killswitch does two things at once: halts trading AND closes all positions. This replaces it with three distinct levels:

- **Level 1 — Pause**: stops new entries, keeps positions open, fully reversible
- **Level 2 — Hard Stop**: stops new entries + cancels pending orders, keeps positions open, fully reversible
- **Level 3 — Emergency Exit**: cancels all orders + closes all positions at market, requires typing “CONFIRM”

The existing `executeKillswitch()` function maps to Level 3 behavior. Levels 1 and 2 are new. Nothing that currently calls `executeKillswitch()` (drawdown monitor, auto-circuit breaker) changes — those always trigger Level 3 because they represent automated risk events. Only the manual UI controls are split into three levels.

**`hardStop(reason: string, userId?: string): Promise<HardStopResult>`**

```typescript
export async function hardStop(reason: string, userId?: string): Promise<{
  success: boolean;
  level: 'HARD_STOP';
  ordersCancelled: number;
  errors: string[];
  isDryRun: boolean;
}> {
  // Activate the existing killswitch flag — this blocks ALL new order execution
  activateKillswitch(reason, 'manual');

  // Also deactivate pause if it was active (hard stop supersedes pause)
  deactivatePause();

  let ordersCancelled = 0;
  const errors: string[] = [];

  // Cancel all PENDING/OPEN orders — but do NOT close positions
  try {
    const openOrders = await getOpenOrders(HL_CONFIG.WALLET_ADDRESS);
    if (openOrders.length > 0) {
      console.warn(`[HL-HARD-STOP] Cancelling ${openOrders.length} open orders`);
      ordersCancelled = await cancelAllOrders(openOrders, userId);
    }
  } catch (err) {
    errors.push(err instanceof Error ? err.message : 'Order cancellation error');
  }

  try {
    await prisma.hyperliquidKillEvent.create({
      data: {
        userId, reason, trigger: 'manual',
        snapshot: { level: 'HARD_STOP', ordersCancelled },
      },
    });
  } catch {}

  console.warn(`[HL-HARD-STOP] Complete — cancelled=${ordersCancelled} errors=${errors.length}`);
  return { success: errors.length === 0, level: 'HARD_STOP', ordersCancelled, errors, isDryRun: HL_CONFIG.DRY_RUN };
}
```

**`resumeTrading(userId?: string): Promise<void>`**

```typescript
// Resumes from EITHER pause or hard stop (but NOT after emergency exit — that requires manual review)
export async function resumeTrading(userId?: string): Promise<void> {
  deactivatePause();
  deactivateKillswitch();
  console.info('[HL] Trading resumed by userId:', userId ?? 'unknown');
}
```

## PHASE 2 — New API Routes

### 2A. Hyperliquid Routes

In `server/src/routes/hyperliquid.ts`, add alongside the existing `/killswitch` routes:

```typescript
import { pauseTrading, hardStop, resumeTrading, executeKillswitch, resetKillswitch } from '../services/hyperliquid/hyperliquidKillswitchService';

// Level 1 — Pause
router.post('/controls/pause', async (req: Request, res: Response) => {
  try {
    const { reason = 'Manual pause' } = req.body;
    const result = await pauseTrading(reason, req.session.userId);
    res.json({ success: true, data: result });
  } catch (err) {
    res.status(500).json({ success: false, error: err instanceof Error ? err.message : 'Pause failed' });
  }
});

// Level 2 — Hard Stop
router.post('/controls/hard-stop', async (req: Request, res: Response) => {
  try {
    const { reason = 'Manual hard stop' } = req.body;
    const result = await hardStop(reason, req.session.userId);
    res.json({ success: true, data: result });
  } catch (err) {
    res.status(500).json({ success: false, error: err instanceof Error ? err.message : 'Hard stop failed' });
  }
});

// Level 3 — Emergency Exit (existing executeKillswitch, just a new route alias)
router.post('/controls/emergency-exit', async (req: Request, res: Response) => {
  try {
    const { reason = 'Emergency exit — manual trigger', confirmText } = req.body;
    // Backend double-check on the confirmation
    if (confirmText !== 'CONFIRM') {
      return res.status(400).json({ success: false, error: 'Must send confirmText: "CONFIRM" to execute emergency exit' });
    }
    const result = await executeKillswitch(reason, 'manual', req.session.userId);
    res.json({ success: true, data: result });
  } catch (err) {
    res.status(500).json({ success: false, error: err instanceof Error ? err.message : 'Emergency exit failed' });
  }
});

// Resume (works from any level)
router.post('/controls/resume', async (req: Request, res: Response) => {
  try {
    await resumeTrading(req.session.userId);
    res.json({ success: true, message: 'Trading resumed' });
  } catch (err) {
    res.status(500).json({ success: false, error: err instanceof Error ? err.message : 'Resume failed' });
  }
});
```

Keep the existing `/killswitch` POST and DELETE routes exactly as they are — they still work as before for backward compatibility and for the drawdown monitor.

## BEHAVIORAL RULES — IMPORTANT

**Pause behavior**:

- Blocks `assertSafe()` in both exchange services — no new orders placed
- Does NOT cancel existing orders
- Does NOT close positions
- Monitoring loop continues running
- Auto-exits from thesis degradation still fire (monitoring doesn’t pause)
- One-click resume

**Hard Stop behavior**:

- Activates the existing killswitch flag (blocks assertSafe)
- Cancels all pending/open orders immediately
- Does NOT close positions
- Monitoring loop continues running
- Auto-exits from thesis degradation are blocked (user has taken manual control)
- One-click resume

**Emergency Exit behavior**:

- Same as the existing `executeKillswitch()` — this is unchanged
- Requires typing “CONFIRM” in the UI
- Requires `confirmText: "CONFIRM"` in the API body
- Closes ALL positions at market
- Cancels ALL orders
- One-click resume still works after this — but positions are already closed

**Automated triggers (drawdown monitor, circuit breaker)**:

- Always trigger Emergency Exit level (existing `executeKillswitch()`)
- These are never downgraded to Pause or Hard Stop — automated risk events always do the full exit
- This behavior is unchanged from the current implementation

**controlLevel derivation**:

- `'HARD_STOP'` if `isKillswitchActive()` is true
- `'PAUSE'` if `isPauseActive()` is true and killswitch is not active
- `'ACTIVE'` if neither is active
- Hard Stop takes precedence over Pause if somehow both are set
