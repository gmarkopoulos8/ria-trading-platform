Expand the existing RIA BOT Replit project.

Goal:
Build a dedicated Daily Scan Engine that ranks and returns the 100 highest-conviction paper-trading opportunities for each session based on combined technical structure, catalyst strength, volatility suitability, and risk-adjusted thesis quality.

This phase should include:
- scheduler logic
- scan orchestration
- database models
- backend APIs
- admin/manual run controls
- daily scan report UI
- ranked top 100 market open dashboard

Core daily scan requirement

Every trading day, the system must generate a ranked list of the 100 highest-conviction paper-trading opportunities for that session using the existing Trade Thesis Engine.

Ranking must be based on a weighted blend of:
- technical structure quality
- catalyst strength
- sentiment direction
- volatility suitability
- liquidity quality
- risk-adjusted thesis quality
- time horizon alignment
- monitoring quality
- clarity of invalidation
- reward-to-risk structure

The system must support:
- market open scan
- optional premarket scan
- manual scan on demand
- historical comparison across daily scans

Scheduler requirements

Build a backend scheduler service for daily scans.

Requirements:
- run a scan every trading day at market open
- support optional premarket scan 30 minutes before market open
- skip weekends
- support market holiday awareness via a pluggable trading-calendar abstraction
- use timezone-aware scheduling
- allow market timezone configuration
- log every scan attempt
- prevent duplicate runs for the same scheduled window
- support retries for partial provider failures
- support partial scan completion if some providers fail

Implementation requirements:
- create a dedicated scheduler service in /server/jobs
- create a scan orchestrator service in /server/services/scans
- isolate scan timing logic from ranking logic
- make scheduling configurable via environment variables or settings table
- support enabling or disabling scheduled scans
- support manual "Run Scan Now" trigger from backend API and admin UI

Suggested scheduler config:
- ENABLE_DAILY_SCANS=true
- MARKET_TIMEZONE=America/New_York
- MARKET_OPEN_HOUR=9
- MARKET_OPEN_MINUTE=30
- PREMARKET_SCAN_ENABLED=true
- PREMARKET_OFFSET_MINUTES=30

Scan universe requirements

Build a scan universe service that selects eligible assets for ranking.

Support:
- liquid stocks

- filtering by:
  - stocks only

The system should:
- define a normalized candidate universe
- fetch candidate symbols
- enrich each symbol with market, technical, catalyst, and risk data
- score all candidates

Allow the scan engine to run in these modes:
- all assets

- conservative only
- aggressive only

Responsibilities:

- persists results

3. dailyRankingService

- applies sorting and tie-breaking

4. dailyMarketReportService
- generates daily summary report
- identifies sector strength/weakness
- identifies highest conviction and highest risk ideas
- builds report payloads for UI

5. scanUniverseService
- fetches and filters candidate universe
- handles stocks and crypto
- supports risk mode filtering

6. scanPersistenceService
- stores runs, results, snapshots, and reports
- supports retrieval by date and run id

Backend API requirements

Build REST endpoints:

1. GET /api/daily-scans
- list scan runs
- filter by date, status, runType, marketSession

2. POST /api/daily-scans/run
- manually trigger a scan
- accept optional body:
  - assetScope
  - riskMode
  - runType
  - marketSession

3. GET /api/daily-scans/:id
- get a single scan run with summary

4. GET /api/daily-scans/:id/results
- get ranked results
- support pagination, sorting, filters

5. GET /api/daily-scans/:id/report
- get full daily market report

6. GET /api/daily-scans/latest
- get latest completed run

7. GET /api/daily-scans/latest/top100
- get latest top 100 ranked opportunities

8. GET /api/daily-scans/compare
- compare two scan dates or run ids

9. GET /api/daily-scans/symbol/:symbol/history
- retrieve historical ranking appearances for a symbol

10. GET /api/daily-scans/settings
- read scheduler settings

11. PATCH /api/daily-scans/settings
- update scheduler settings
